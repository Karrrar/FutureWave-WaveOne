# Terminal Basics for Mac

This guide covers essential command line commands for macOS using the built-in Terminal and Linux-style commands.

## Getting Started

### How to Open Terminal on Mac:

- **Spotlight**: `Cmd + Space` ‚Üí Type "Terminal" ‚Üí Press Enter
- **Finder**: Applications ‚Üí Utilities ‚Üí Terminal
- **Launchpad**: Find Terminal in Utilities folder
- **VS Code Terminal**: `Ctrl + `` (backtick) in VS Code
- **Dock**: Add Terminal to Dock for quick access

## Essential Commands

### 1. `pwd` - Print Working Directory

Shows your current location in the file system.

```bash
pwd
```

**Example Output:**
```
/Users/yourusername/Documents
```

**What it does:**
- Displays the full path of your current directory
- Helps you understand where you are in the file system
- Essential for navigation and file operations

---

### 2. `cd` - Change Directory

Navigate between folders.

```bash
# Go to a specific folder
cd Documents
cd "My Projects"

# Go back one level
cd ..

# Go back two levels
cd ../..
```

**Examples:**
```bash
cd Desktop             # Go to Desktop folder
cd "Creative Projects" # Use quotes for spaces
cd ..                 # Go to parent directory
cd ~/Downloads        # Go directly to Downloads
```

---

### 3. `cd ~` - Go to Home Directory

Quickly navigate to your user home folder.

```bash
cd ~
```

**What it does:**
- Takes you to `/Users/yourusername`
- `~` is a shortcut for your home directory
- Same as typing `cd /Users/yourusername`

**Alternative:**
```bash
cd          # Just 'cd' with no arguments also goes home
```

---

### 4. `cd /` - Go to Root Directory

Navigate to the system root.

```bash
cd /
```

**What it shows:**
- Takes you to the filesystem root
- You'll see system directories like: `bin`, `etc`, `usr`, `var`
- This is where the entire macOS filesystem starts

---

### 5. `ls` - List Files and Directories

Show contents of current directory.

```bash
# Basic listing
ls

# Detailed listing with permissions, size, date
ls -l

# Show hidden files too (files starting with .)
ls -a

# Combine options for detailed view with hidden files
ls -la

# Human-readable file sizes
ls -lh
```

**Example Output:**
```bash
ls -la
total 24
drwxr-xr-x   5 user  staff   160 Nov 22 10:30 .
drwxr-xr-x  20 user  staff   640 Nov 22 09:15 ..
-rw-r--r--   1 user  staff  1024 Nov 22 10:25 .gitignore
drwxr-xr-x   3 user  staff    96 Nov 22 10:30 Documents
-rw-r--r--   1 user  staff   256 Nov 22 10:20 README.md
```

**Useful variations:**
```bash
ls -lt        # Sort by modification time (newest first)
ls -lS        # Sort by file size
ls *.txt      # List only .txt files
```

---

### 6. `mkdir` - Make Directory

Create new folders.

```bash
# Create single directory
mkdir my-project

# Create multiple directories at once
mkdir folder1 folder2 folder3

# Create nested directories (parent directories too)
mkdir -p projects/web/my-site/src
```

**Examples:**
```bash
mkdir "New Project"           # Create folder with spaces
mkdir project-2024           # Create project folder
mkdir -p ~/Code/JavaScript   # Create nested in home
mkdir {frontend,backend}     # Create multiple using braces
```

---

### 7. `clear` - Clear Terminal Screen

Clean up your terminal display.

```bash
clear
```

**Alternative methods:**
- **Keyboard shortcut**: `Cmd + K` (clears scrollback too)
- **Keyboard shortcut**: `Ctrl + L` (standard clear)

**Difference:**
- `clear` - clears visible area
- `Cmd + K` - clears entire scrollback buffer

---

### 8. `cat` - Display File Contents

View the contents of text files.

```bash
# View file contents
cat filename.txt

# View multiple files in sequence
cat file1.txt file2.txt

# Create a simple file (type content, then Ctrl+D to save)
cat > newfile.txt

# Append to a file
cat >> existing-file.txt
```

**Examples:**
```bash
cat package.json              # View package.json content
cat ~/.bash_profile          # View bash profile
cat *.md                     # View all markdown files
cat /etc/hosts              # View system hosts file
```

**Useful for:**
- Quick file viewing
- Combining files
- Creating simple text files
- Checking configuration files

---

### 9. `Ctrl+C` - Cancel Current Command

Stop a running command or process.

**How to use:**
- Press and hold `Ctrl`, then press `C`
- Universal interrupt signal
- Works in all terminal applications

**When to use:**
```bash
# Stop long-running commands
ping google.com       # Press Ctrl+C to stop

# Cancel file creation
cat > somefile.txt    # Press Ctrl+C to cancel

# Stop development servers
yarn dev             # Press Ctrl+C to stop dev server
```

**What it does:**
- Sends SIGINT (interrupt) signal to process
- Gracefully stops most applications
- Returns control to terminal
- Essential for process management

**Note:** Some processes may require `Ctrl+Z` (suspend) or `kill` command

---

### 10. `code .` - Open Current Directory in VS Code

Open the current directory in Visual Studio Code.

```bash
# Open current directory in VS Code
code .

# Open specific file
code filename.txt

# Open specific directory
code my-project

# Create and open new file
code newfile.js

# Open with specific options
code . --new-window     # Open in new window
code . --reuse-window   # Reuse existing window
```

**Prerequisites:**
- Visual Studio Code installed
- Command line tools enabled in VS Code
- Run `Cmd+Shift+P` ‚Üí "Shell Command: Install 'code' command in PATH"

**Examples:**
```bash
cd ~/Code/my-project  # Navigate to project
code .               # Open entire project

code package.json    # Open specific file
code src/            # Open src directory
code README.md       # Open README file
```

**Mac-specific features:**
```bash
# Open current directory in Finder
open .

# Open VS Code from Finder
# Right-click folder ‚Üí Services ‚Üí "Open in Visual Studio Code"

# Using Spotlight
# Cmd+Space ‚Üí type "code" ‚Üí select VS Code
```

**Troubleshooting:**
- If `code` command doesn't work:
  1. Open VS Code
  2. `Cmd+Shift+P` ‚Üí "Shell Command: Install 'code' command in PATH"
  3. Restart terminal
- Alternative: Drag folder to VS Code in Dock

## Mac-Specific Features

### VS Code Integrated Terminal:

VS Code's built-in terminal is excellent for development on Mac:

**Open VS Code Terminal:**
- **Keyboard**: `Ctrl + `` (backtick) or `Cmd + `` 
- **Menu**: Terminal ‚Üí New Terminal
- **Command Palette**: `Cmd + Shift + P` ‚Üí "Terminal: Create New Terminal"

**Configure Default Shell:**
```json
// In VS Code settings.json
{
  "terminal.integrated.defaultProfile.osx": "zsh",
  // or
  "terminal.integrated.defaultProfile.osx": "bash"
}
```

**Mac-specific Terminal Shortcuts:**
```bash
Cmd + `          # Toggle terminal (alternative)
Ctrl + `         # Standard toggle
Cmd + Shift + ` # Create new terminal
Cmd + \         # Split terminal
Cmd + 1/2/3     # Switch between terminals
```

**Development Workflow:**
```bash
# Open VS Code in project
code my-project

# Terminal opens in project automatically
pwd              # Shows: /Users/name/my-project
ls -la          # List project files
yarn dev       # Start development server
```

### Spotlight Integration:
```bash
# Open files in default applications
open file.txt           # Open in TextEdit
open .                  # Open current directory in Finder
```


### Useful Mac Paths:
```bash
cd ~/Desktop              # Desktop folder

```

- `‚Üë` / `‚Üì` - Navigate command history

## Common Scenarios

### Set Up Development Environment:
```bash
cd ~                      # Go home
mkdir Code               # Create Code directory
cd Code                  # Enter Code directory
mkdir my-first-project   # Create project folder
cd my-first-project      # Enter project
pwd                      # Confirm location
```

### Explore a Downloaded Project:
```bash
cd ~/Downloads           # Go to Downloads
ls -lt                   # List files by date
cd project-folder        # Enter the project
ls -la                   # See all files including hidden
cat README.md           # Read the documentation
```

### Quick File Management:
```bash
pwd                      # Check current location
ls                       # See what's here
mkdir backup            # Create backup folder
cat important.txt       # Check file contents
clear                   # Clean up terminal
```
