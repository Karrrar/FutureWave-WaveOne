# Terminal Basics for Windows

This guide covers essential command line commands for Windows using Linux-style commands in PowerShell, Command Prompt, or Git Bash.

## Getting Started

### How to Open Terminal on Windows:

- **PowerShell**: `Windows + X` ‚Üí Select "Windows PowerShell" or "Terminal"
- **Git Bash**: Install Git for Windows ‚Üí Right-click ‚Üí "Git Bash Here"
- **Windows Terminal**: `Windows + R` ‚Üí Type `wt` ‚Üí Press Enter
- **VS Code Terminal**: `Ctrl + `` (backtick) in VS Code
- **Start Menu**: Search "PowerShell" or "Git Bash"

## Essential Commands

### 1. `pwd` - Print Working Directory

Shows your current location in the file system.

```bash
pwd
```

**Example Output:**
```
C:\Users\YourName\Documents
```

**What it does:**
- Displays the full path of your current directory
- Helps you understand where you are in the file system
- Useful when you're lost in directories

---

### 2. `cd` - Change Directory

Navigate between folders.

```bash
# Go to a specific folder
cd Documents
cd "My Projects"

# Go back one level
cd ..

# Go back two levels
cd ../..
```

**Examples:**
```bash
cd Desktop          # Go to Desktop folder
cd "Program Files"   # Use quotes for spaces
cd ..              # Go to parent directory
```

---

### 3. `cd ~` - Go to Home Directory

Quickly navigate to your user home folder.

```bash
cd ~
```

**What it does:**
- Takes you to `C:\Users\YourName`
- `~` is a shortcut for your home directory
- Works the same way across all operating systems

---

### 4. `cd /` - Go to Root Directory

Navigate to the system root.

```bash
cd /
```

**Windows Note:**
- **PowerShell**: Goes to drive root (C:\)
- **Git Bash**: Uses Unix-style paths (/c, /d)

---

### 5. `ls` - List Files and Directories

Show contents of current directory.

**Git Bash / Linux-style:**
```bash
# Basic listing
ls

# Detailed listing with permissions, size, date
ls -l

# Show hidden files too
ls -a

# Combine options
ls -la
```

**PowerShell alternatives:**
```powershell
# Basic listing (ls works but is an alias)
ls
Get-ChildItem

# Show hidden files
ls -Force
Get-ChildItem -Force

# Detailed listing
ls -l  # This WON'T work in PowerShell!
Get-ChildItem | Format-List

```


### Drive Navigation:
```bash
# Switch between drives
cd /c          # Go to C: drive
cd /d          # Go to D: drive

# In Command Prompt:
C:             # Switch to C: drive
D:             # Switch to D: drive
```

**Example Output (PowerShell):**
```
    Directory: C:\Users\YourName\Documents

Mode                 LastWriteTime         Length Name
----                 -------------         ------ ----
d-----        11/22/2025   2:30 PM                Documents
d-----        11/22/2025   1:20 PM                Downloads  
-a----        11/22/2025   3:15 PM           1024 file.txt
-a----        11/22/2025   2:45 PM           2048 project.zip
```

**Best approach for Windows:**
- **Use Git Bash** for Unix-style commands (`ls -la` works perfectly)
- **Use PowerShell native commands** for better Windows integration
- **Avoid Command Prompt** for development work

---

### 6. `mkdir` - Make Directory

Create new folders.

```bash
# Create single directory
mkdir my-project

# Create multiple directories
mkdir folder1 folder2 folder3

# Create nested directories
mkdir -p projects/web/my-site
```

**Examples:**
```bash
mkdir "New Folder"     # Create folder with spaces
mkdir project-2024     # Create project folder
mkdir -p a/b/c        # Create nested structure
```

---

### 7. `clear` - Clear Terminal Screen

Clean up your terminal display.

```bash
clear
```

**Windows Options:**
- **PowerShell**: `clear`, `cls`, or `Clear-Host`
- **Git Bash**: `clear`

**Keyboard Shortcut:**
- `Ctrl + L` (works in most terminals)

---

### 8. `cat` - Display File Contents

View the contents of text files.

```bash
# View file contents
cat filename.txt

# View multiple files
cat file1.txt file2.txt

# Create a simple file
cat > newfile.txt
# Type content, then Ctrl+C to save
```

**Windows Note:**
- **PowerShell**: `cat` works, or use `Get-Content`
- **Git Bash**: `cat` works perfectly

**Examples:**
```bash
cat package.json       # View package.json content
cat README.md         # View README file
cat *.txt            # View all .txt files
```

---

### 9. `Ctrl+C` - Cancel Current Command

Stop a running command or process.

**How to use:**
- Press and hold `Ctrl`, then press `C`
- Works in all Windows terminals
- Immediately stops the current operation

**When to use:**
```bash
# If a command is taking too long
ping google.com       # Press Ctrl+C to stop

# If you made a mistake while typing
cat > somefile.txt    # Press Ctrl+C to cancel without saving

# To stop development servers
yarn dev            # Press Ctrl+C to stop the server
```

**What it does:**
- Sends interrupt signal to running process
- Returns you to the command prompt
- Doesn't save partial work (be careful!)
- Essential for controlling long-running processes

---

### 10. `code .` - Open Current Directory in VS Code

Open the current directory in Visual Studio Code.

```bash
# Open current directory in VS Code
code .

# Open specific file
code filename.txt

# Open specific directory
code my-project

# Create and open new file
code newfile.js
```

**Prerequisites:**
- Visual Studio Code must be installed
- VS Code must be added to PATH during installation
- Or manually add VS Code to system PATH

**Examples:**
```bash
cd my-project         # Navigate to project
code .               # Open entire project in VS Code

code package.json    # Open specific file
code src/            # Open src directory
```

## Windows-Specific Tips

### VS Code Integrated Terminal:

VS Code has a built-in terminal that's perfect for development work:

**Open VS Code Terminal:**
- **Keyboard**: `Ctrl + `` (backtick)
- **Menu**: Terminal ‚Üí New Terminal
- **Command Palette**: `Ctrl + Shift + P` ‚Üí "Terminal: Create New Terminal"

**Choose Your Shell in VS Code:**
```json
// In VS Code settings.json
{
  "terminal.integrated.defaultProfile.windows": "Git Bash",
  // or
  "terminal.integrated.defaultProfile.windows": "PowerShell"
}
```

**VS Code Terminal Shortcuts:**
```bash
Ctrl + `         # Toggle terminal
Ctrl + Shift + ` # Create new terminal
Ctrl + \         # Split terminal
Ctrl + PageUp/Down # Switch between terminals
```

**Example Workflow:**
```bash
# Open VS Code in project folder
code my-project

# Terminal automatically opens in project
# Press Ctrl + ` if terminal isn't visible
pwd              # Shows: /c/Users/Name/my-project
ls               # Lists project files
yarn dev         # Start development server
```

### PowerShell vs Git Bash:

| Feature        | Git Bash                     | PowerShell                           |
| -------------- | ---------------------------- | ------------------------------------ |
| `ls -la`       | ‚úÖ Works                      | ‚ùå Use `Get-ChildItem -Force`         |
| `ls -l`        | ‚úÖ Works                      | ‚ùå Use `Get-ChildItem \| Format-List` |
| `cat`          | ‚úÖ Works                      | ‚úÖ Works (alias)                      |
| `clear`        | ‚úÖ Works                      | ‚úÖ Works                              |
| Unix paths     | ‚úÖ `/c/Users`                 | ‚ùå `C:\Users`                         |
| Cross-platform | ‚úÖ Same commands on Mac/Linux | ‚ùå Windows-specific                   |



### Path Differences:
- **Windows paths**: `C:\Users\Name\Documents`
- **Unix-style paths**: `/c/Users/Name/Documents` (Git Bash)
- **PowerShell**: Accepts both styles

## Common Scenarios

### Development Workflow with VS Code:
```bash
# Open project in VS Code
code my-project          # VS Code opens with integrated terminal

# Terminal is already in project folder
pwd                      # Shows: /c/path/to/my-project
ls                       # See project files
cat package.json         # Check project config
yarn install            # Install dependencies
yarn dev                # Start development
```

### Navigate to a Project:
```bash
cd ~                    # Go home
cd Documents           # Go to Documents
mkdir my-projects      # Create projects folder
cd my-projects         # Enter projects folder
pwd                    # Confirm location
```

### Explore a Directory:
```bash
ls -la                 # List everything with details
cat README.md          # Read the README
cd src                 # Go to source folder
ls                     # See what's inside
```

### Clean Workspace:
```bash
clear                  # Clean terminal
pwd                    # Check where you are
ls                     # See current contents
```
