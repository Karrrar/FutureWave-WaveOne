# Terminal Basics for Linux

This guide covers essential command line commands for Linux distributions. These are the native commands that work across all Linux systems.

## Getting Started

### How to Open Terminal on Linux:

- **Ubuntu/GNOME**: `Ctrl + Alt + T`
- **KDE**: `Ctrl + Alt + T` or find "Konsole" in menu
- **Applications Menu**: Look for "Terminal" or "Console"
- **Right-click**: Right-click on desktop ‚Üí "Open Terminal"
- **VS Code Terminal**: `Ctrl + `` (backtick) in VS Code
- **GUI**: Activities ‚Üí Search "Terminal"

## Essential Commands

### 1. `pwd` - Print Working Directory

Shows your current location in the file system.

```bash
pwd
```

**Example Output:**
```
/home/username/Documents
```

**What it does:**
- Displays the absolute path of your current directory
- Essential for understanding your location in the filesystem
- Helps when writing scripts or troubleshooting

---

### 2. `cd` - Change Directory

Navigate between directories.

```bash
# Go to a specific directory
cd Documents
cd /var/log

# Go back one level
cd ..

# Go back two levels
cd ../..

# Go to previous directory
cd -
```

**Examples:**
```bash
cd Desktop              # Go to Desktop folder
cd "My Projects"        # Use quotes for spaces
cd ..                  # Go to parent directory
cd /etc/nginx          # Go to nginx config
cd -                   # Return to previous location
```

---

### 3. `cd ~` - Go to Home Directory

Quickly navigate to your user home folder.

```bash
cd ~
```

**What it does:**
- Takes you to `/home/username`
- `~` is a shell expansion for your home directory
- Same as typing your full home path

**Variations:**
```bash
cd ~           # Go to your home
cd ~otheruser  # Go to another user's home (if permitted)
cd             # Just 'cd' also goes home
```

---

### 4. `cd /` - Go to Root Directory

Navigate to the filesystem root.

```bash
cd /
```

**What you'll find:**
- System directories: `bin`, `etc`, `home`, `usr`, `var`
- This is where the entire Linux filesystem hierarchy starts
- All absolute paths start from here

**Important directories from root:**
```bash
/home      # User home directories
/etc       # System configuration files
/var       # Variable data (logs, databases)
/usr       # User programs and libraries
/bin       # Essential system binaries
```

---

### 5. `ls` - List Files and Directories

Show contents of current directory.

```bash
# Basic listing
ls

# Long format with details
ls -l

# Show hidden files (starting with .)
ls -a

# Combine options
ls -la

# Human-readable file sizes
ls -lh

# Sort by modification time
ls -lt

# Recursive listing
ls -R
```

**Example Output:**
```bash
ls -la
total 28
drwxr-xr-x 3 user user 4096 Nov 22 10:30 .
drwxr-xr-x 5 user user 4096 Nov 22 09:15 ..
-rw-r--r-- 1 user user  220 Nov 22 10:25 .bashrc
-rw-r--r-- 1 user user  807 Nov 22 10:25 .profile
drwxr-xr-x 2 user user 4096 Nov 22 10:30 Documents
-rw-r--r-- 1 user user 1024 Nov 22 10:20 README.txt
```

**Understanding permissions:**
- `drwxr-xr-x`: Directory with read, write, execute for owner
- `-rw-r--r--`: File with read/write for owner, read-only for others

**Useful variations:**
```bash
ls -1          # One file per line
ls --color     # Colorized output
ls *.txt       # List only .txt files
ls -d */       # List only directories
```

---

### 6. `mkdir` - Make Directory

Create new directories.

```bash
# Create single directory
mkdir my-project

# Create multiple directories
mkdir dir1 dir2 dir3

# Create parent directories as needed
mkdir -p path/to/deep/directory

# Create with specific permissions
mkdir -m 755 public-folder
```

**Examples:**
```bash
mkdir "Project Files"              # Directory with spaces
mkdir -p ~/Code/Python/my-app     # Create nested structure
mkdir backup-$(date +%Y%m%d)      # Create with date
mkdir {frontend,backend,docs}     # Create multiple using brace expansion
```

---

### 7. `clear` - Clear Terminal Screen

Clean up your terminal display.

```bash
clear
```

**Alternative methods:**
- `Ctrl + L` - Keyboard shortcut (fastest)
- `reset` - Complete terminal reset
- `printf '\033[2J\033[H'` - Raw escape sequence

**When to use:**
- Clean up cluttered terminal
- Before starting new tasks
- When taking screenshots
- During presentations

---

### 8. `cat` - Display File Contents

View and manipulate text files.

```bash
# Display file contents
cat filename.txt

# Display multiple files
cat file1.txt file2.txt

# Create a new file
cat > newfile.txt
# Type content, then Ctrl+D to save

# Append to existing file
cat >> existing-file.txt

# Show line numbers
cat -n filename.txt

# Show non-printing characters
cat -A filename.txt
```

**Examples:**
```bash
cat /etc/passwd            # View system users
cat ~/.bashrc              # View bash configuration
cat package.json           # View project configuration
cat /var/log/syslog | tail # View recent system logs

```

**Useful for:**
- Quick file viewing
- Combining files
- Creating simple text files
- Piping to other commands

---

### 9. `Ctrl+C` - Cancel Current Command

Send interrupt signal to stop running processes.

**How to use:**
- Press and hold `Ctrl`, then press `C`
- Sends SIGINT signal to foreground process
- Most common way to stop commands

**When to use:**
```bash
# Stop long-running commands
ping google.com       # Press Ctrl+C to stop
find / -name "*.txt"  # Press Ctrl+C if taking too long

# Cancel file operations
cat > somefile.txt    # Press Ctrl+C to cancel without saving

# Stop development servers
yarn dev            # Press Ctrl+C to stop server

# Cancel package installations
sudo apt update      # Press Ctrl+C to cancel
```

**What it does:**
- Sends SIGINT (Signal Interrupt) to process
- Process can handle signal gracefully or terminate
- Returns control to shell
- Essential for process control

**Related signals:**
- `Ctrl+Z` - Suspend process (can be resumed)
- `Ctrl+\` - Send SIGQUIT (stronger than Ctrl+C)
- `kill -9 PID` - Force kill process

### 10. `code .` - Open Current Directory in VS Code

Open the current directory in Visual Studio Code editor.

```bash
# Open current directory in VS Code
code .

# Open specific file
code filename.txt

# Open specific directory
code /path/to/project

# Create and open new file
code newfile.js

# Open multiple files
code file1.txt file2.txt

# Open with options
code . --new-window     # Force new window
code . --reuse-window   # Reuse existing window
code . --wait          # Wait for VS Code to close
```

**Prerequisites:**
- Visual Studio Code installed
- VS Code in system PATH


**Examples:**
```bash
cd ~/Code/my-project  # Navigate to project
code .               # Open entire project

code package.json    # Open specific file
code src/            # Open src directory
code README.md       # Open README

# Development workflow
mkdir my-app && cd my-app  # Create and enter directory
npm init -y                # Initialize project
code .                     # Open in VS Code
```


### Useful Environment Variables:

## Advanced Usage

### Combining Commands:
```bash
cd ~ && pwd && ls           # Chain commands with &&
ls -la | grep ".txt"         # Pipe to filter .txt files
cat file1.txt file2.txt > combined.txt  # Redirect output
```

### Wildcards and Patterns:
```bash
ls *.txt           # All .txt files
ls file?.txt       # file1.txt, file2.txt, etc.
ls [abc]*.txt      # Files starting with a, b, or c
mkdir test{1..5}   # Create test1, test2, test3, test4, test5
```

### Command History:
```bash
history            # Show command history
!!                 # Run last command
!n                 # Run command number n from history
!string            # Run last command starting with 'string'
Ctrl + R           # Search command history interactively
```
